# obMenu

Небольшая панель форматирования для тех, кто живет в Markdown.

[English](README.md) | [Русский](README.ru.md) | [Español](README.es.md) | [Deutsch](README.de.md)

obMenu - clean-room плагин с Markdown-панелью инструментов, вдохновленный идеей cMenu. Он держит часто используемые действия рядом с редактором: заголовки, чекбоксы, callout-блоки, ссылки, подсветку, код, списки, жирный, курсив и несколько спокойных помощников.

Закрепите панель снизу, показывайте ее рядом с выделением или перетащите в удобное место. Меньше походов в command palette, меньше запоминания шорткатов, больше письма.

> Статус: готов к GitHub-релизам. Подача в каталог community plugins: в процессе.

## Что уже работает

### Markdown-действия

Встроенный реестр команд поддерживает:

- Жирный
- Курсив
- Зачеркнутый
- Подчеркнутый
- Подсветку
- Очистку форматирования
- Инлайн-код
- Блок кода
- Цитату
- Callout
- Чекбокс
- Маркированный список
- Нумерованный список
- Markdown-ссылку
- Wikilink
- Заголовки H1-H6

Панель по умолчанию не показывает все команды сразу. Она начинается с полезного набора для письма, а остальные действия доступны через пресеты или вкладку настроек.

### Заголовки

Панель по умолчанию показывает H1, H2, H3 и H4 отдельными кнопками. H5, H6 и сгруппированное меню заголовков тоже доступны, если нужна другая раскладка.

### Размещение панели

obMenu поддерживает четыре режима размещения:

- `fixed`: держать панель снизу рабочей области.
- `selection`: показывать рядом с выделенным текстом.
- `cursor`: размещать рядом с текущим выделением или курсором, когда браузер дает usable selection rectangle.
- `manual`: перетащить панель за ручку и сохранить позицию.

Режимы selection, cursor и manual удерживают панель внутри viewport даже в небольших окнах.

### Настройка панели

Панель редактируется в настройках:

- Добавить любую встроенную команду, которой еще нет на панели.
- Удалить ненужные кнопки.
- Добавить визуальные разделители между группами.
- Переставить кнопки drag and drop.
- Переставить кнопки стрелками, если drag and drop неудобен.
- Сбросить панель к набору по умолчанию.

Разделители являются настоящими элементами панели, поэтому можно сделать группы вместо одной длинной линии кнопок.

### Пресеты

Пресеты помогают быстро начать:

- `Writer`: полный набор по умолчанию для обычного Markdown-письма.
- `Zettelkasten`: заголовки, чекбоксы, цитаты, подсветка, wikilinks и Markdown-ссылки.
- `Code notes`: инлайн-код, блоки кода, списки, callout-блоки и ссылки.
- `Compact`: более короткая панель со сгруппированными заголовками и компактным стилем.

После выбора пресета можно добавлять и удалять кнопки, добавлять разделители и менять порядок. Пресеты не блокируют панель, а только дают стартовую раскладку.

### Стили

- `default`: обычная плотность панели в стиле Obsidian.
- `compact`: кнопки меньше, когда заметке нужно больше места.

Оба стиля используют theme variables Obsidian, поэтому obMenu следует светлым и темным темам без собственной цветовой системы.

### Детали редактирования

- Действия панели возвращают фокус в активный Markdown-редактор.
- Пустые инлайн-действия ставят курсор между вставленными маркерами.
- Если выделенный текст уже обернут тем же инлайн-форматированием, повторное нажатие снимает обертку.
- Переключение чекбокса сохраняет отступы и обрабатывает пустые checked/unchecked task items.
- Callout превращает текущую строку в `> [!note]` или убирает существующий callout marker.
- Очистка форматирования снимает распространенные инлайн-обертки с выделения или текущей строки.

### Безопасность настроек

Сохраненные настройки нормализуются при загрузке. Если появятся старые или некорректные данные, obMenu вернется к значениям по умолчанию вместо поломки панели.

### Поддержка релизов

В репозитории есть локальная сборка релиза и tag-based GitHub release workflow. Релизные assets включают `main.js`, `manifest.json` и `styles.css`, с GitHub artifact attestations.

## Настройки

Вкладка настроек позволяет:

- Включить или выключить панель.
- Выбрать режим размещения.
- Выбрать визуальный стиль.
- Сбросить ручную позицию панели.
- Применить пресеты Writer, Zettelkasten, Code notes и Compact.
- Добавить встроенные кнопки.
- Добавить визуальные разделители.
- Переставить кнопки drag and drop или стрелками.
- Удалить кнопки.
- Сбросить элементы панели к набору по умолчанию.

## Установка из GitHub release

Скачайте эти файлы из последнего GitHub release:

```text
main.js
manifest.json
styles.css
```

Положите их в vault:

```text
.obsidian/plugins/obmenu
```

Затем включите `obMenu` в настройках community plugins.

## Локальная сборка

Соберите release folder:

```bash
npm run build:dist
```

Скопируйте папку:

```text
dist/obmenu
```

в vault:

```text
.obsidian/plugins/obmenu
```

Затем включите `obMenu` в настройках community plugins.

## Разработка

Установить зависимости:

```bash
npm install
```

Запустить тесты:

```bash
npm run test
```

Запустить TypeScript-проверку:

```bash
npm run typecheck
```

Собрать root plugin bundle:

```bash
npm run build
```

Собрать release folder:

```bash
npm run build:dist
```

Запустить полный pre-release check:

```bash
npm run check
```

## Privacy

obMenu работает локально внутри Obsidian. Он не отправляет заметки, выделения, настройки или телеметрию.

## Inspiration

obMenu вдохновлен cMenu, более ранним плагином панели форматирования для Obsidian. Этот проект не копирует исходные файлы cMenu.
